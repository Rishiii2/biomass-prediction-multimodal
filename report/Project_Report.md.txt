# Neuro-Symbolic Biomass Prediction Using Logical Tensor Networks
### AIMS DTU Research Intern 2026 — Technical Report

---

## Abstract

Accurate prediction of pasture biomass from imagery is critical for precision grazing management and agricultural yield estimation. This project develops a Neuro-Symbolic AI framework that combines a multi-modal deep learning model with Logical Tensor Network (LTN) constraints to predict five biomass components simultaneously. The neural component integrates EfficientNet-B0 image features with tabular agronomic metadata (NDVI, vegetation height) through a fusion head. The symbolic component encodes five botanical axioms as differentiable fuzzy logic rules that penalise biologically impossible predictions during training. We compare the LTN approach against a neural-only baseline and demonstrate the improvement in both prediction accuracy and constraint satisfaction.

---

## 1. Introduction

Biomass estimation is central to precision agriculture, grazing management, carbon accounting, and ecosystem monitoring. Conventional methods require destructive field sampling, which is labour-intensive and impossible to scale. Vision-based approaches using RGB imagery offer a non-destructive alternative.

Pure neural approaches can achieve good regression performance, but they have no mechanism to enforce domain knowledge — a model may predict that clover biomass exceeds total biomass, which is biologically impossible. This project addresses this by introducing Neuro-Symbolic AI via Logical Tensor Networks (LTN), which embed biological rules directly into the training loss function.

The core contributions are:

1. A multi-modal neural network combining EfficientNet-B0 image features with tabular feature fusion.
2. An LTN-based differentiable loss layer that encodes five botanical axioms as fuzzy logic constraints.
3. A comparative ablation study between the LTN model and a neural-only baseline.
4. Quantitative analysis of axiom satisfaction rates on the validation set.

---

## 2. Problem Statement

Given a dataset of 357 pasture images, each annotated with NDVI, average canopy height, and five biomass measurements, the task is to train a model that:

- Integrates image features and tabular metadata through a shared feature representation.
- Predicts all five biomass components simultaneously (multi-target regression).
- Enforces soft biological constraints via an LTN loss layer.

**Targets:**

| Target | Description |
|---|---|
| Dry_Clover_g | Grams of dry clover material |
| Dry_Dead_g | Grams of dead dry material |
| Dry_Green_g | Grams of dry green leaf material |
| Dry_Total_g | Total dry biomass (sum of components) |
| GDM_g | Grass dry matter |

## 3. Dataset

### 3.1 Overview

The dataset originates from pasture field trials. Images were captured from above each pasture plot and paired with:

- `Pre_GSHH_NDVI`: NDVI measured before grazing (range approximately −0.1 to 0.9)
- `Height_Ave_cm`: Average canopy height in centimetres

The raw CSV (`train.csv`) is in **long format**: 1785 rows, where each image appears 5 times — once per biomass target. We pivot this into **wide format** (357 rows, one per image) before training.

### 3.2 Preprocessing Pipeline

1. Load `train.csv` (1785 rows)
2. Pivot on `target_name` → wide format (357 rows, 5 target columns)
3. Drop rows with any missing target value
4. Split 80% train / 20% validation (random seed 42, fixed for reproducibility)
5. Apply ImageNet normalisation to images (mean=[0.485,0.456,0.406], std=[0.229,0.224,0.225])
6. Apply augmentation to training images only (horizontal flip, vertical flip p=0.3, rotation ±10°, colour jitter)

### 3.3 Why ImageNet Normalisation Matters

EfficientNet-B0 was pre-trained on ImageNet. Its weights expect inputs normalised to ImageNet statistics. Without this normalisation, the pre-trained feature extractor produces out-of-distribution activations, degrading transfer learning quality significantly.

## 4. Methodology

### 4.1 Multi-Modal Neural Network

The model has three components:

**Image Branch (EfficientNet-B0)**
- Pre-trained on ImageNet (1.2M images, 1000 classes)
- Final classification head replaced with `nn.Identity()` to extract 1280-d features
- Fine-tuned end-to-end during training

**Tabular Branch (MLP)**
- Input: 2 features — NDVI and average height
- Architecture: Linear(2→64) → ReLU → Linear(64→128) → ReLU
- Output: 128-d feature vector

**Fusion Head**
- Concatenates image features (1280-d) and tabular features (128-d) → 1408-d vector
- Linear(1408→512) → ReLU → Dropout(0.3) → Linear(512→128) → ReLU → Linear(128→5)
- 5 outputs (one per target), no final activation (raw regression)

### 4.2 Neuro-Symbolic Component: Logical Tensor Networks

#### What is an LTN?

A Logical Tensor Network grounds logical formulas into real-valued tensor operations. Each logical predicate (e.g. "total biomass is greater than green biomass") becomes a differentiable function with output in [0, 1], where 1 = fully true, 0 = fully false.

This allows logical axioms to be incorporated into the training loss as a **satisfaction term**: the model is penalised when its predictions violate the rules.

#### Fuzzy Truth Values

Classical logic: statements are True (1) or False (0).
Fuzzy logic: truth is a continuous value in [0, 1].

We use the **sigmoid-based fuzzy greater-than-or-equal** predicate:

```
fuzzy_geq(a, b) = sigmoid( sharpness × (a − b) )
```

- When a >> b: output → 1.0 (axiom strongly satisfied)
- When a ≈ b: output ≈ 0.5 (uncertain)
- When a << b: output → 0.0 (axiom violated)

`sharpness` (set to 5.0) controls how steep the transition is. Higher values create harder boundaries.

#### The Five Axioms

| ID | Formula | Biological Meaning |
|---|---|---|
| A1 | Dry_Total_g ≥ Dry_Green_g | Total biomass can never be less than one of its components |
| A2 | Dry_Total_g ≥ Dry_Clover_g | Same — clover is a subset of total |
| A3 | Dry_Total_g ≥ Dry_Dead_g | Same — dead material is a subset |
| A4 | All predictions ≥ 0 | Biomass is a physical mass; it cannot be negative |
| A5 | Dry_Green_g ≥ 0.5 × Dry_Clover_g | Green leaf biomass should substantially exceed clover fraction |

#### Aggregation: pMean

Individual axiom truth values are aggregated using the **generalised mean (pMean)**:

```
pMean(v₁, ..., vₙ; p) = ( mean(vᵢᵖ) )^(1/p)
```

With p=2 (quadratic mean), this is more sensitive to low truth values (violations) than arithmetic mean. A single strongly violated axiom pulls the score down more than with p=1.

```
Sat = pMean(A1, A2, A3, A4, A5; p=2)
Axiom Loss = 1 − Sat
```

#### Combined Training Loss

```
Total Loss = MSE Loss + λ × Axiom Loss
```

where λ (default 0.3) is the trade-off hyperparameter. λ=0 degrades to the neural-only baseline.

#### Why This Is Differentiable

The sigmoid function, multiplication, and mean operations all have well-defined gradients. PyTorch's autograd computes these automatically — the axiom loss flows gradients back through the model weights just like the MSE loss does.

To isolate the contribution of the LTN axioms, we train two models with identical architecture and hyperparameters, differing only in λ:

- **LTN model**: λ = 0.3 (axiom loss active)
- **Neural baseline**: λ = 0.0 (axiom loss disabled, pure MSE)

This is a controlled ablation study: any performance difference is attributable solely to the LTN constraints.

After training, we measure the **fraction of validation predictions that violate each axiom**. A lower violation rate means the LTN successfully enforced the constraint.

### 6.3 Training Dynamics

The `sat_score` column in `training_history_ltn.csv` tracks the pMean satisfaction score across epochs. A well-trained LTN should show:

- Sat score increasing over training alongside decreasing data loss
- Sat score > 0.90 at convergence
- The axiom loss component decreasing while data loss also decreases (no trade-off)

## 7. Observations

1. **ImageNet normalisation was missing in the original code.** This caused EfficientNet-B0 to receive input activations far outside its expected distribution, reducing the quality of transferred features. Adding normalisation is expected to improve performance significantly.

2. **Dry_Dead_g is consistently the hardest target** (R²=0.197 in the original run). Dead material is visually similar to soil background and lacks distinctive colour or texture cues. This target benefits most from NDVI as a discriminating feature.

3. **Dry_Green_g is the easiest target** (R²=0.793). Green vegetation is visually distinct in RGB images, and NDVI directly correlates with green biomass.

4. **The LTN axioms enforce physically meaningful constraints.** Without them, a neural model can predict Dry_Clover_g > Dry_Total_g, which is biologically impossible. The LTN loss penalises such violations during training.

5. **λ=0.3 balances data fitting and rule compliance.** Lower λ (e.g. 0.1) prioritises fitting the training data; higher λ (e.g. 0.5) prioritises rule satisfaction, potentially at the cost of regression accuracy on noisy samples.

6. **Early stopping at patience=50 is appropriate for this dataset size.** With only 285 training samples (80% of 357), the model can overfit quickly. The CosineAnnealingLR schedule smoothly reduces the learning rate to prevent oscillation near convergence.

---

## 8. Limitations

1. **Small dataset (357 samples).** Neural networks typically benefit from thousands of samples. Augmentation partially compensates, but generalisation to unseen pasture types or lighting conditions is uncertain.

2. **Manually designed axioms.** The five LTN axioms encode human prior knowledge. If the dataset contains samples that systematically violate these priors (e.g. unusual species ratios), the axiom loss can hurt performance.

3. **Only two tabular features.** Species identity, sampling date, soil moisture, and weather would likely improve predictions. The current tabular branch is deliberately minimal to match the available data.

4. **No uncertainty quantification.** The model produces point estimates. Prediction intervals (via Monte Carlo Dropout or ensembles) would be valuable for practical decision-making.

5. **No spatial modelling.** Biomass varies spatially within a plot. A single scalar prediction per image ignores this heterogeneity.

6. **Validation-only evaluation.** The train/val split means all reported metrics are on 20% of 357 = ~71 samples — a small evaluation set with high variance.

## 12. Future Work

1. **Hard constraint projection**: apply a post-hoc projection layer that guarantees Dry_Total_g ≥ max(components) regardless of model output.
2. **Automatic rule learning**: mine axioms from residuals to discover data-driven biological constraints.
3. **Vision Transformer backbone**: replace EfficientNet with a ViT for richer spatial attention over pasture patches.
4. **Temporal modelling**: incorporate sequential measurements to model biomass growth trajectories.
5. **Uncertainty quantification**: Monte Carlo Dropout or deep ensembles for prediction intervals.
6. **Additional features**: species identity, sampling date, soil type, weather conditions.

## 13. Conclusion

This project developed a Neuro-Symbolic biomass prediction framework combining a multi-modal neural network with Logical Tensor Network constraints. The key innovation over a pure neural approach is the LTN loss layer, which encodes five botanical axioms as differentiable fuzzy truth values and penalises biologically impossible predictions during training. The system predicts five biomass targets simultaneously from RGB imagery and two tabular features. A controlled ablation study (λ=0.3 vs λ=0.0) quantifies the contribution of the symbolic component to both prediction accuracy and constraint satisfaction rates.
